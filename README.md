## v5.7 手機答題操作修正

- 手機進入答題頁後，自動隱藏大型頁首、統計區、存檔區與主導覽列。
- 答題頁改為接近全螢幕顯示，讓題目與選項保留更多空間。
- 「下一題」按鈕固定在手機畫面底部，避開安全區並提高點擊層級。
- 送出答案後會自動把「下一題」帶入可視範圍並取得操作焦點。
- 尚未送出答案時，重新選擇與確定送出按鈕會固定在內容底部附近。
- 非答題頁的上方導覽改為單列橫向滑動，不再排列成多排。
- 針對較矮的手機螢幕進一步縮小題目上方間距。

## v5.7 圖鑑圖片模式與響應式畫面

- 「圖鑑收錄／全部」改為怪獸圖片牆，只顯示怪獸圖像、名稱與淨化狀態。
- 圖鑑收錄不再顯示題目內容。

## v5.7 可點擊圖鑑中心

- 待破解怪獸、已淨化怪獸、圖鑑收錄三張統計卡片都可點擊。
- 點擊後會直接切換並顯示相對應的怪獸內容。
- 新增搜尋功能，可依怪獸名稱、弱點線索、題目與主題查找。
- 新增圖鑑完成率與視覺化進度條。
- 統計卡加入滑鼠懸停、按壓與鍵盤焦點效果。

## v5.7 怪獸圖鑑與獎勵規則

- 單元不限首次挑戰，只要當次 10 題全部答對，就獲得 1 枚金幣。
- 主線答對獲得 10 EXP；答錯不獲得 EXP。
- 答錯題目會化身為環保怪獸，加入「怪獸弱點圖鑑」。
- 圖鑑卡片顯示怪獸名稱、圖示、弱點線索、題目與破解狀態。
- 點擊「再次挑戰」時不主動顯示解析。
- 弱點複習答對後標示為「已淨化／已掌握」，獲得 5 EXP，不發放金幣。
- 弱點複習答錯時才顯示解析，仍不公布正確答案，也不給 EXP 或金幣。

## 直接遊玩

下載專案後，開啟 `index.html` 即可遊玩。遊戲不需要伺服器或資料庫。

## 發布到 GitHub Pages

1. 在 GitHub 建立新的 repository。
2. 將這個資料夾內的所有檔案上傳到 repository 根目錄。
3. 開啟 repository 的 **Settings → Pages**。
4. 在 **Build and deployment** 選擇 **Deploy from a branch**。
5. Branch 選擇 `main`，資料夾選擇 `/ (root)`，再按 **Save**。
6. 等候 GitHub 顯示發布網址。
